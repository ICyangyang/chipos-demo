## Consistency Check

- [x] **子模块 FP 覆盖顶层 FP**: PASS — Uart_tx_fifo 的 FP-F01~FP-F08 覆盖顶层 FP-001~FP-008；TX_Control_FSM 的 FP-S01~FP-S10 覆盖顶层 FP-009~FP-028
- [x] **FIFO 深度与计数器位宽一致**: PASS — 深度 16，count 为 5-bit (0-16)，可表示 0~16 共 17 个值
- [x] **bit_counter 位宽与波特率分频一致**: PASS — bit_counter 为 4-bit (0-15)，每 bit 16 个 PCLK 周期
- [x] **LCR 数据位宽编码完整**: PASS — LCR[1:0] 覆盖 5/6/7/8 四种数据位宽
- [x] **LCR 校验类型编码完整**: PASS — LCR[5:4] 覆盖偶/奇/固定1/固定0 四种校验
- [x] **端口方向与子模块连接一致**: PASS — 顶层输入连接子模块输入，子模块输出连接顶层输出
- [ ] **LCR[6] 和 LCR[7] 定义缺失**: FAIL — LCR 为 8-bit，但仅定义了 [5:0]，[7:6] 未定义
- [ ] **1.5 停止位与 2 停止位的区分条件不完整**: FAIL — LCR[2]=1 时 Spec 仅说 "1.5 or 2 stop bits"，未给出明确区分条件 (标准 UART 中 1.5 stop 仅用于 5-bit 数据，但 Spec 未显式声明)
- [ ] **LCR 动态切换行为未定义**: FAIL — FP-027 行为未定义，Spec 仅列为待验证点但未给出预期行为
- [ ] **enable 暂停粒度未定义**: FAIL — Spec 未说明 enable=0 时是立即暂停还是等待当前 bit 发送完成后再暂停
- [ ] **sub-8-bit 数据的 tx_buffer 位映射未定义**: FAIL — 当数据位宽为 5/6/7 时，tx_buffer[7:0] 中哪些位对应有效数据位未明确

## Ambiguities

| ID | Item | Question | Suggested Resolution |
|----|------|----------|---------------------|
| A1 | LCR[7:6] 未定义 | LCR 为 8-bit 宽，但 Spec 仅定义了 [5:0]。LCR[7:6] 的功能是什么？是否应保留为 0 或作为其他配置？ | 建议明确 LCR[7:6] 为保留位 (Reserved)，读返回 0，写忽略；或定义其功能 (如 Break Control / DLAB 等，参考 16550 标准) |
| A2 | 1.5 vs 2 停止位区分 | LCR[2]=1 时，Spec 写 "1.5 or 2 stop bits"，但未给出区分条件。何时发 1.5 停止位，何时发 2 停止位？ | 建议遵循标准 UART 规范：当数据位宽=5 时为 1.5 停止位，数据位宽>=6 时为 2 停止位。需在 Spec 中显式声明 |
| A3 | LCR 动态切换 | 当 tx_state!=IDLE 时修改 LCR，发送行为如何？是使用帧开始时的 LCR 快照还是实时采样？ | 建议在帧开始 (IDLE→START) 时锁存 LCR 到内部寄存器 lcr_latch，整个帧期间使用 lcr_latch，忽略 LCR 端口变化 |
| A4 | enable 暂停粒度 | enable=0 时，bit_counter 立即冻结还是等当前 bit 发完？如果立即冻结，恢复后从断点继续是否会导致位内毛刺？ | 建议立即冻结 bit_counter (与 Spec 4.2 节描述一致)，恢复后从断点继续。需确认 TXD 在暂停期间是否保持当前电平 |
| A5 | sub-8-bit 数据位映射 | 当 LCR[1:0]=2'b00 (5-bit) 时，tx_buffer[7:0] 中哪 5 位是有效数据？是 tx_buffer[4:0] 还是其他映射？ | 建议有效数据始终从 tx_buffer 的 LSB 开始对齐，即 5-bit 模式使用 tx_buffer[4:0]，tx_buffer[7:5] 忽略 |
| A6 | FIFO 满时 push 的错误指示 | 当 tx_fifo_full=1 且 tx_fifo_push=1 时，除了忽略写入外，是否需要产生错误指示信号或中断？ | 建议增加 tx_fifo_overflow 输出信号 (pulse)，或在 Spec 中明确声明 "满时写入静默忽略，无错误指示" |
| A7 | FIFO pop 时序 | FIFO 的 pop 请求与 rd_data 的时序关系：是组合逻辑 Read-Through (rd_data 在 pop 前已稳定) 还是寄存器输出？ | Spec 4.1 节写 "data_out = data_fifo[op_count]" 为组合逻辑，建议明确声明为 Read-Through 模式，rd_data 在 pop 当周期即可采样 |
